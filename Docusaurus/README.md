# Docusaurus Documentation Tool

Docusaurus is React Based Static site generator which runs on node.js

### Command to install for typescript based project
```bash
npx create-docusaurus@latest my-website classic --typescript
```

### To run
```bash
cd my-website
npm run start
```

---

### Docusaurus is a modern static website generator, so we need to build the website into a directory of static contents and put it on a web server so that it can be viewed. To build the website:

```bash
npm run build
```

## For documentation of the TypeScript files

Install this:
```bash
npm install --save-dev typedoc typedoc-plugin-markdown
```

Next create **typedoc.json** file in your root folder and add the following:

```json
{
  "entryPoints": ["../src"],
  "out": "docs/api",
  "plugin": ["typedoc-plugin-markdown"],
  "readme": "none",
  "excludePrivate": true,
  "excludeProtected": false,
  "hideGenerator": true
}
```

Here the entry point is for the path of the folder from which the files are selected for documentation and `out` is the destination folder.

Then in **sidebars.ts** add this:

```ts
const sidebars: SidebarsConfig = {
  apiSidebar: [
    {
      type: 'autogenerated',
      dirName: '.',   // auto-picks up docs/api/*
    },
  ],
};
```

Now in `src` folder make a ts file (`index.ts`) and include this:

```ts
/**
 * Adds two numbers together.
 *
 * @param a - The first number
 * @param b - The second number
 * @returns The sum of `a` and `b`
 *
 * @example
 * ```ts
 * const result = add(2, 3); // 5
 * ```
 */
export function add(a: number, b: number): number {
  return a + b;
}

/**
 * Represents a user in the system.
 */
export interface User {
  /** Unique identifier */
  id: string;
  /** Display name */
  name: string;
  /** Optional email address */
  email?: string;
}
```

#### To generate documentation run this command:

```bash
npx typedoc --out docs/api src/index.ts
```

The above command will generate the markdown files in `docs/api`.

### Now in docusaurus.config.ts

In navbar items add:

```ts
themeConfig: {
  navbar: {
    title: 'My TS Docs',
    items: [
      // add this
      { type: 'docSidebar', sidebarId: 'apiSidebar', label: 'API', position: 'left' },
    ],
  },
} satisfies Preset.ThemeConfig,
```

## For Python documentation

You will have another folder for the python project. There, install pydoc-markdown:

```bash
uv add pydoc-markdown
```

Now let's say you have a folder containing `.py` files which need to be documented. To generate the documentation run this command:

```bash
pydoc-markdown -p name_of_folder > docs/python-doc
```

Breaking it down:
- **`-p`** represents that it is a package
- **`name_of_folder`** which contains the `.py` files (folder path can also be given)
- **`>`** copy to
- **`docs/python-doc`** — create a target folder in the `docs` folder of the Docusaurus project, then add the path perfectly for the redirection

Now the same procedure as that of the TypeScript portion — in `sidebar.ts` add the code and in the navbar of `docusaurus.config.ts` add that item.

> **Note:** All the above documentation works on Markdown files.

## For OpenAPI.json Documentation

Install the Scalar plugin:

```bash
npm install @scalar/docusaurus
```

Now in `docusaurus.config.ts` add this in plugins:

```js
'@scalar/docusaurus',
{
  label: 'API Reference',
  routePath: '/api-reference',
  configuration: {
    url: '/openapi/open-api.json',
    theme: 'purple',
    hideModels: true,
    // to disable the AI feature
    agent: {
      disabled: true,
    },
    // to remove the MCP server section
    mcp: {
      disabled: true,
    },
  },
},
```
