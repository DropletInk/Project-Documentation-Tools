# Docusaurus Documentation Tool

Docusaurus is React Based Static site generator which runs on node.js

### Command to install for typescript based project
```bash
npx create-docusaurus@latest my-website classic --typescript
```

### To run
```bash
cd my-website
npm run start
```

---

### Docusaurus is a modern static website generator, so we need to build the website into a directory of static contents and put it on a web server so that it can be viewed. To build the website:

```bash
npm run build
```

## For documentation of the TypeScript files

Install this:
```bash
npm install --save-dev typedoc typedoc-plugin-markdown
```

Next create **typedoc.json** file in your root folder and add the following:

This contains entrypoints of which documentation will be done , and the destination folder where the doumentation files (markdown files ) will be stored

```json
{
  "entryPoints": [  
    "./"
  ],
  "exclude": [
    "**/build/**",
    "**/dist/**",
    "**/*.test.ts",
    "**/*.spec.ts",
    "**/tests/**",
    "**/node_modules/**",
    "./code-documentation/**",
    "./.docusaurus/**",
    "./docker-compose*",
    "**/jest.config.ts"
  ],
  
  "entryPointStrategy": "expand", 
  "tsconfig": "tsconfig.json",
  "out": "code-documentation/docs/api", 
  "plugin": [
    "typedoc-plugin-markdown"
  ],

  "readme": "none", 
  "excludePrivate": true, 
  "excludeExternals": true, 
  "hideGenerator": true, 
  "outputFileStrategy": "modules",
  
  "disableSources": true,     
  "sort": [
    "source-order"   
  ],

  "navigation": {
    "includeCategories": false,  
    "includeGroups": false,
    "includeFolders": true
  }
}
```

Here the entry point is for the path of the folder from which the files are selected for documentation and `out` is the destination folder.

Then in **sidebars.ts** add this:


**The sidebar.ts needs to be set as it will orgainse your markdown files at the frontend level**
### Example for Autogeneration where the docusaurus itself identifies the struture of the targeted folder to organise
```ts
const sidebars: SidebarsConfig = {
  apiSidebar: [
    {
      type: 'autogenerated',
      dirName: '.',   // auto-picks up docs/api/* 
    },
  ],
};
```
### Example for Manually Organise the generated markdown files
```ts
apiSidebar: [
    // ... your other top-level links like 'backend'
    {
      type: 'category',
      label: 'controller',
      items: [
        'api/controller/jest.config',
        {
          type: 'category',
          label: 'src',
          items: [
            {
              type: 'category',
              label: 'config',
              // This makes clicking the folder open the index file
              link: { type: 'doc', id: 'api/controller/src/config/index' },
              items: [
                // This forces "index" to explicitly display inside the folder menu
                'api/controller/src/config/index',
                'api/controller/src/config/database',
                'api/controller/src/config/env',
                'api/controller/src/config/handlers.config',
                'api/controller/src/config/permission',
              ],
            },
            {
              type: 'category',
              label: 'core',
              items: [{ type: 'autogenerated', dirName: 'api/controller/src/core' }],
            },
            {
              type: 'category',
              label: 'module',
              items: [{ type: 'autogenerated', dirName: 'api/controller/src/module' }],
            },
            'api/controller/src/server',
          ],
        },
      ],
    },
    {
      type: 'category',
      label: 'backend',
      items: [{ type: 'autogenerated', dirName: 'api/backend' }],
    },
    {
      type: 'category',
      label: 'packages',
      items: [{ type: 'autogenerated', dirName: 'api/packages' }],
    },
    // ... your other links like 'packages'
  ],
```
Now in `src` folder make a ts file (`index.ts`) and include this:

#### In Case of the index.ts file to be included in documentation  you need to write the  following on the top of every index file 
For Example 
**@module packages/models/index**

```ts
/**
 /**
 * ---
 * sidebar_label: config
 * ---
 * @module controller/src/config/index    
 *
 */
Other Contents of the index.ts file 
```

#### To generate documentation run this command:


```bash
npx typedoc --out docs/api src/index.ts
```
here `docs/api` is the Destination folder path 
`src/index.ts` is the source file path 

The above command will generate the markdown files in `docs/api`.

### Now in docusaurus.config.ts

In navbar items add: 
```ts
themeConfig: {
  navbar: {
    title: 'My TS Docs',
    items: [
      // add this
      { type: 'docSidebar', sidebarId: 'apiSidebar', label: 'API', position: 'left' },
    ],
  },
} satisfies Preset.ThemeConfig,
```

## For Python documentation

You will have another folder for the python project. There, install pydoc-markdown:

```bash
uv add pydoc-markdown
```

Now let's say you have a folder containing `.py` files which need to be documented. To generate the documentation run this command:

```bash
pydoc-markdown -p name_of_folder > docs/python-doc
```

Breaking it down:
- **`-p`** represents that it is a package
- **`name_of_folder`** which contains the `.py` files (folder path can also be given)
- **`>`** copy to
- **`docs/python-doc`** — create a target folder in the `docs` folder of the Docusaurus project, then add the path perfectly for the redirection

Now the same procedure as that of the TypeScript portion — in `sidebar.ts` add the code and in the navbar of `docusaurus.config.ts` add that item.

> **Note:** All the above documentation works on Markdown files.

## For OpenAPI.json Documentation

Install the Scalar plugin:

```bash
npm install @scalar/docusaurus
```

Now in `docusaurus.config.ts` add this in plugins:

```js
'@scalar/docusaurus',
{
  label: 'API Reference',
  routePath: '/api-reference',
  configuration: {
    url: '/openapi/open-api.json',
    theme: 'purple',
    hideModels: true,
    // to disable the AI feature
    agent: {
      disabled: true,
    },
    // to remove the MCP server section
    mcp: {
      disabled: true,
    },
  },
},
```
